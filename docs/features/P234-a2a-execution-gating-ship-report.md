## 1. Summary

P234 separates A2A message routing from execution to prevent background token burn.
Previously, the A2A dispatcher auto-spawned Claude-backed worktrees upon message delivery.
Now, routing (message_ledger, trust checks, transition_queue) is preserved but explicit
execution intent is required before any Claude-backed spawn.

## 4. What Changed

### Before
- A2A delivery to claude/* recipients auto-invoked agent-spawner.ts:spawnAgent()
- Every inbound A2A message could trigger a worktree spawn (token cost)
- Trust gate contained hardcoded claude agent identities

### After
- deliverToAgent() sends pg_notify to worktree agents (no spawn)
- Virtual agents get transition_queue entries only
- trustGate() uses only ["system", "gary"] as non-agent authorities
- Dispatcher marks messages as dispatched (read_at) after delivery
- All log lines use [deliver]/[notify] terminology

## 5. Risk Assessment

**Low risk.** This is a subtraction — removing auto-spawn behavior. The routing layer
(message_ledger, trust checks, transition_queue) is unchanged. The only behavioral change
is that Claude-backed agents must now be explicitly spawned rather than auto-spawned on
message receipt.
