## Critical Bugs Found

### Bug 1: `fn_enqueue_mature_proposals()` Case Mismatch (P204)
**File**: `database/ddl/013-gate-pipeline-wiring.sql`
**Root cause**: The function's CASE statement uses `'Draft'`, `'Review'`, `'Develop'`, `'Merge'` (mixed case) but the database stores `'DRAFT'`, `'REVIEW'`, `'DEVELOP'`, `'MERGE'` (uppercase). The CASE never matches, so the function always hits CONTINUE and never enqueues any proposals.
**Impact**: The gate pipeline service (`hermes-gate-pipeline`) runs but does ZERO work. No proposals are ever automatically advanced.
**Fix needed**:
```sql
-- Change CASE values to uppercase:
CASE rec.status
  WHEN 'DRAFT'   THEN v_gate := 'D1'; v_to_state := 'REVIEW';
  WHEN 'REVIEW'  THEN v_gate := 'D2'; v_to_state := 'DEVELOP';
  WHEN 'DEVELOP' THEN v_gate := 'D3'; v_to_state := 'MERGE';
  WHEN 'MERGE'   THEN v_gate := 'D4'; v_to_state := 'COMPLETE';
  WHEN 'TRIAGE'  THEN v_gate := 'T1'; v_to_state := 'FIX';
  WHEN 'FIX'     THEN v_gate := 'F1'; v_to_state := 'DEPLOYED';
  ELSE CONTINUE;
END CASE;
```
Also: remove hardcoded `'claude/one'` from spawn metadata.

### Bug 2: `prop_create` SQL Error (P205)
**File**: `src/infra/postgres/proposal-storage-v2.ts` line 243
**Root cause**: `FILTER (WHERE ws.stage_order = MIN(ws.stage_order) OVER ())` â€” PostgreSQL does not allow window functions inside FILTER clauses.
**Impact**: ALL `prop_create` MCP calls fail. Must insert proposals directly via SQL.
**Fix needed**: Rewrite query to avoid window functions in FILTER:
```sql
-- Instead of:
SELECT MIN(ws.stage_name) FILTER (WHERE ws.stage_order = MIN(ws.stage_order) OVER ()) AS start_stage
-- Use:
SELECT (ARRAY_AGG(ws.stage_name ORDER BY ws.stage_order))[1] AS start_stage
```

### Bug 3: Missing Gate Evaluator (P206)
**Root cause**: The orchestrator dispatches agents on state-change notifications but no agent evaluates mature proposals and calls `transition_proposal`. The gate pipeline was supposed to do this but is broken (Bug 1). Even if fixed, it dispatches cubics but those agents don't automatically transition proposals.
**Impact**: Proposals get stuck at "mature" indefinitely.
**Fix needed**: Add gate-evaluator to AGENT_DISPATCH for each state, or fix `fn_enqueue_mature_proposals()` to directly transition proposals (not just enqueue).

### Bug 4: `transition_proposal` Requires `decided_by`
The MCP tool requires a `decided_by` parameter (not documented in earlier calls). The value must be a registered agent in `agent_registry` (FK constraint). Using `gate-agent` works.

### Bug 5: `prop_transition` Tool Error
`prop_transition` throws `Cannot read properties of undefined (reading 'toUpperCase')` â€” different from `transition_proposal` which works with correct params.
