### Behavior
The `agent_retire` tool:
1. Looks up agent by ID in in-memory map
2. Releases all active claims if `releaseClaims` !== false
3. Sets agent status to "offline"
4. Records reason in `agent.lastError`
5. Returns confirmation text

## Recommendation

P175 should be **reopened to DEVELOP** or **split into a follow-up proposal** to implement the missing components. The current `agent_retire` tool is a minimal viable retirement but does not achieve the proposal's core goal: "preserving institutional knowledge before agent shutdown."

A potential approach:
1. Create `retirement_plan` + `knowledge_transfer_log` tables
2. Extend `retireAgent()` to call knowledge extraction before shutdown
3. Add `extract_knowledge` MCP tool that reads `agent_memory` then writes to `knowledge_entries`
4. Add `onboard_successor` MCP tool that queries `knowledge_entries` by predecessor context
5. Wire audit trail to `knowledge_transfer_log`
