## Summary

Deploy the missing `roadmap.agent_health` table and wire pulse monitoring to restore fleet observability. The `pulse_fleet` MCP tool currently fails with "relation roadmap.agent_health does not exist."

## Problem Statement

15 agents are registered but health monitoring is non-functional:
- `pulse_fleet` returns error: `relation "roadmap.agent_health" does not exist`
- No heartbeat tracking for active agents
- No degraded/unhealthy detection
- Fleet dashboard shows nothing

## Proposed Solution

### 1. Agent Health Table
```sql
CREATE TABLE roadmap.agent_health (
    id              int8 GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    agent_identity  text NOT NULL REFERENCES roadmap.agent_registry(identity),
    status          text NOT NULL DEFAULT 'unknown'
        CHECK (status IN ('healthy', 'degraded', 'unhealthy', 'unknown', 'offline')),
    last_heartbeat  timestamptz DEFAULT now(),
    heartbeat_ms    int,                    -- latency of last heartbeat
    cpu_pct         numeric(5,2),           -- if available
    memory_mb       int,                    -- if available
    active_leases   int DEFAULT 0,
    error_count     int DEFAULT 0,          -- errors since last healthy
    last_error      text NULL,
    checked_at      timestamptz DEFAULT now(),
    metadata        jsonb DEFAULT '{}'
);

CREATE INDEX idx_agent_health_identity ON roadmap.agent_health (agent_identity);
CREATE INDEX idx_agent_health_status ON roadmap.agent_health (status) WHERE status != 'healthy';
```

### 2. Heartbeat Tracking View
```sql
CREATE VIEW roadmap.v_fleet_health AS
SELECT
    ar.identity,
    ar.role,
    ar.status AS registration_status,
    ah.status AS health_status,
    ah.last_heartbeat,
    ah.active_leases,
    ah.error_count,
    CASE
        WHEN ah.last_heartbeat < now() - interval '5 minutes' THEN 'stale'
        WHEN ah.last_heartbeat < now() - interval '1 minute' THEN 'lagging'
        ELSE 'current'
    END AS heartbeat_freshness
FROM roadmap.agent_registry ar
LEFT JOIN LATERAL (
    SELECT * FROM roadmap.agent_health
    WHERE agent_identity = ar.identity
    ORDER BY checked_at DESC LIMIT 1
) ah ON true;
```

### 3. Auto-Offline Detection
```sql
-- Mark agents as offline if no heartbeat for 10 minutes
CREATE OR REPLACE FUNCTION fn_detect_offline_agents()
RETURNS void LANGUAGE sql AS $$
    INSERT INTO roadmap.agent_health (agent_identity, status, checked_at)
    SELECT identity, 'offline', now()
    FROM roadmap.agent_registry ar
    WHERE ar.status = 'active'
      AND NOT EXISTS (
          SELECT 1 FROM roadmap.agent_health ah
          WHERE ah.agent_identity = ar.identity
            AND ah.last_heartbeat > now() - interval '10 minutes'
      );
$$;
```
