## Summary

Implement a self-healing Directed Acyclic Graph (DAG) system for proposal dependencies that automatically detects, reports, and resolves dependency conflicts, cycles, and stale relationships.

## Problem Statement

Current dependency tracking (`proposal_dependencies`) has basic cycle detection but lacks:
1. Automatic detection of stale dependencies
2. Self-healing mechanisms for broken dependency chains
3. Visual dependency graph representation
4. Conflict resolution strategies

## Proposed Solution

### 1. Enhanced Dependency Tracking
```sql
ALTER TABLE roadmap.proposal_dependencies
    ADD COLUMN dependency_type text DEFAULT 'hard' 
        CHECK (dependency_type IN ('hard', 'soft', 'advisory')),
    ADD COLUMN health_status text DEFAULT 'healthy'
        CHECK (health_status IN ('healthy', 'stale', 'broken', 'circular')),
    ADD COLUMN last_verified timestamptz DEFAULT now(),
    ADD COLUMN auto_resolved bool DEFAULT false;
```

### 2. Self-Healing Mechanisms
- **Stale Dependency Detection:** Flag dependencies not updated in 30+ days
- **Circular Dependency Resolution:** Automatic breaking of circular dependencies
- **Broken Chain Repair:** Suggest alternative dependency paths
- **Health Monitoring:** Continuous dependency graph health checks

### 3. Visual Dependency Graph
- Interactive DAG visualization
- Real-time dependency health status
- Impact analysis for proposed changes
- Critical path identification
