## Summary

Add performance metrics and health monitoring for the 90+ MCP tool surface. Currently there is zero visibility into tool reliability, latency, or error rates.

## Problem Statement

AgentHive exposes 90+ MCP tools with no instrumentation:
- No execution time tracking
- No error rate monitoring
- No per-tool success/failure metrics
- No way to identify flaky or broken tools
- Agent-native-capabilities warns: "MCP breadth can overwhelm agents unless exposure is scoped"

## Proposed Solution

### 1. Tool Metrics Table
```sql
CREATE TABLE roadmap.mcp_tool_metrics (
    id              int8 GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    tool_name       text NOT NULL,
    agent_identity  text NOT NULL,
    execution_ms    int NOT NULL,
    success         bool NOT NULL,
    error_class     text NULL,           -- 'timeout', 'validation', 'internal', 'auth'
    error_message   text NULL,
    input_size      int,                 -- bytes of input
    output_size     int,                 -- bytes of output
    called_at       timestamptz DEFAULT now()
);

CREATE INDEX idx_mcp_metrics_tool ON roadmap.mcp_tool_metrics (tool_name, called_at);
CREATE INDEX idx_mcp_metrics_errors ON roadmap.mcp_tool_metrics (tool_name) WHERE NOT success;
```

### 2. Tool Health Summary View
```sql
CREATE VIEW roadmap.v_tool_health AS
SELECT
    tool_name,
    COUNT(*) AS total_calls,
    COUNT(*) FILTER (WHERE NOT success) AS error_count,
    ROUND(100.0 * COUNT(*) FILTER (WHERE NOT success) / NULLIF(COUNT(*), 0), 1) AS error_rate_pct,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY execution_ms) AS p50_ms,
    PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY execution_ms) AS p95_ms,
    PERCENTILE_CONT(0.99) WITHIN GROUP (ORDER BY execution_ms) AS p99_ms,
    MAX(called_at) AS last_called
FROM roadmap.mcp_tool_metrics
WHERE called_at > now() - interval '24 hours'
GROUP BY tool_name
ORDER BY error_count DESC, p95_ms DESC;
```

### 3. Role-Based Tool Filtering
```sql
CREATE TABLE roadmap.agent_tool_access (
    id              int8 GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    agent_role      text NOT NULL,
    tool_pattern    text NOT NULL,       -- glob pattern: 'prop_*', 'pulse_*'
    access_level    text DEFAULT 'full' CHECK (access_level IN ('full', 'read_only', 'none')),
    UNIQUE (agent_role, tool_pattern)
);
```

### 4. Metrics Collection Hook
Add to MCP server middleware:
```typescript
// In MCP server tool dispatch
const start = Date.now();
try {
  const result = await handler(args);
  await recordMetric(toolName, agentId, Date.now() - start, true);
  return result;
} catch (error) {
  await recordMetric(toolName, agentId, Date.now() - start, false, error.class);
  throw error;
}
```
