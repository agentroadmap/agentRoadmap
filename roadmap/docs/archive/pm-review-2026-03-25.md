## 2. What Changed This Week

### Major Strategic Shift: "Product Development Platform" (STATE-087, 88, 89)

The product is **repositioning** from "project management for agents" to "agent-native product development platform." This is significant:

| Old Vision | New Vision (Cubic Architecture) |
|------------|--------------------------------|
| Agents manage roadmaps | Agents build products autonomously |
| Human writes requirements | PM orchestrates expert agent teams |
| File-based collaboration | Docker-sandboxed "cubics" per phase |
| Single-agent workflow | Multi-agent specialized teams |

**Cubic Architecture Key Design:**
- **Design Cubic** → Build Cubic → Test Cubic → Ship Cubic
- Phase gates: G1 (Design Review) → G2 (Build Complete) → G3 (Test Passed) → G4 (Ship Approved)
- PM role: **Orchestrator only** — routes work, monitors flow, triggers gates
- LLM routing: Design=Opus, Build/Test=Sonnet, Ship=Haiku
- Cross-cubic comms via Postgres message table
